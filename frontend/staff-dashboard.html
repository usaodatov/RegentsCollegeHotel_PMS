Staff Dashboard

NOTES (demo + coursework)

[1] Why simplified: Max 25 reservations only (5 rooms × 5 days). Enough to show full flow, not overbuild for demo, i think ok.

[2] How data come here: Page fetch data as JSON from backend (GET). JS render rows after load, sort by arrival date then name, pretty simple way.

[3] Cancel flow: Click Cancel → confirm Yes/No. Yes send POST to backend. After cancel, list fetched again so screen stay fresh, no stale view.

[4] Data storage: Backend talk to RDS MySQL. Create/cancel update DB rows. UI never touch DB direct, just display, no magic.

[5] Future evolution: Later can add Check-in, Check-out, Billing, Invoice, Room move. Same pattern: endpoint + button + refresh list, should work.
